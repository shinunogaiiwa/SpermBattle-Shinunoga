# 端口占用问题解决方案

## 当前情况
- **端口 3000**: 被进程 PID 23752 (node.exe) 占用
- **端口 3001**: 被进程 PID 32912 (node.exe) 占用

这些通常是 Next.js 前端开发服务器。

## 解决方案

### 方法 1: 停止占用端口的进程（推荐）

#### 使用 PowerShell 脚本：
```powershell
.\stop-ports.ps1
```

#### 手动停止：
```powershell
# 停止端口 3000 的进程
Stop-Process -Id 23752 -Force

# 停止端口 3001 的进程
Stop-Process -Id 32912 -Force
```

#### 使用命令行：
```cmd
taskkill /PID 23752 /F
taskkill /PID 32912 /F
```

### 方法 2: 使用不同的端口

如果这些服务是你需要的，可以修改前端配置使用其他端口：

#### 在 `frontend/package.json` 中修改：
```json
{
  "scripts": {
    "dev": "next dev -p 3002"
  }
}
```

#### 或者在启动时指定端口：
```bash
cd frontend
npm run dev -- -p 3002
```

### 方法 3: 检查并继续使用现有服务

如果这些服务已经在运行，你可以直接访问：
- http://localhost:3000
- http://localhost:3001

查看它们是否是你要使用的应用。

## 检查端口占用的命令

```powershell
# 查看端口 3000 的占用情况
netstat -ano | findstr ":3000"

# 查看端口 3001 的占用情况
netstat -ano | findstr ":3001"

# 查看所有 Node.js 进程
Get-Process node

# 查看特定进程的详细信息
Get-Process -Id 23752
```

## 预防措施

在启动新的开发服务器之前，先检查端口是否被占用：
```powershell
# 检查端口是否被占用
$port = 3000
$connection = Get-NetTCPConnection -LocalPort $port -ErrorAction SilentlyContinue
if ($connection) {
    Write-Host "端口 $port 已被占用" -ForegroundColor Red
} else {
    Write-Host "端口 $port 可用" -ForegroundColor Green
}
```
